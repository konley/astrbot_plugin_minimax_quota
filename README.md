# astrbot_plugin_minimax_quota

AstrBot 插件：查询 MiniMax 套餐限额，支持指令查询、定时播报、低额度阶梯告警。

## 功能特性

- **指令查询** `/mmquota`：随时查询各模型当前时段剩余额度，信号灯样式展示。
- **管理员限制**（默认关）：可设置仅 AstrBot 框架管理员才能调用指令。
- **定时播报**（默认关）：按 Cron 表达式定时把完整限额播报推送给指定的人或群。
- **阶梯告警**（默认关）：后台定期轮询，余量每跌破一个档位（如 20% → 10% → 0%）提醒一次，不会重复打扰。
- **全部参数暴露 WebUI**：无需改代码，在插件配置面板即可设置。

## 安装

将本插件目录放入 AstrBot 的 `data/plugins/`，或在 WebUI 插件市场用仓库地址安装：

```
https://github.com/konley/astrbot_plugin_minimax_quota
```

依赖会自动安装（`httpx`、`apscheduler`）。安装后在 **WebUI → 插件配置** 填入 MiniMax API Key 即可。

## 播报样式

```
📊 MiniMax 限额播报
🟡 general｜本时段 76% · 3.4 小时后重置（20:00）
🟢 video  ｜本时段 100% · 8.1 小时后重置（06-30 00:00）
```

- 信号灯：🟢 = 100% 充足，🟡 = 21~99% 偏紧，🔴 = ≤20% 告急。
- 重置时间同时给出相对时间和绝对时间；跨天的绝对时间会带日期。

阶梯告警样式：

```
⚠️ MiniMax 限额播报 - 额度告警
🔴 general 本时段仅剩 18%（已跌破 20% 档）· 1.2 小时后重置（20:00）
请留意用量。
```

## 配置项说明（WebUI）

| 配置 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `api_key` | string | 空 | **必填**。MiniMax API Key（Bearer Token） |
| `api_base` | string | 官方地址 | 限额查询接口地址，一般无需修改 |
| `timeout` | int | 30 | 请求超时（秒） |
| `report_title` | string | `MiniMax 限额播报` | 播报/查询结果的标题，可自定义 |
| `admin_only` | bool | false | 仅管理员可调用 `/mmquota` |
| `schedule_enable` | bool | false | 启用定时全量播报 |
| `schedule_cron` | string | `0 9 * * *` | 定时播报 Cron 表达式，见下文 |
| `monitor_enable` | bool | false | 启用低额度阶梯告警 |
| `monitor_interval` | int | 30 | 后台轮询间隔（分钟），最小 1 |
| `alert_levels` | string | `20,10,0` | 告警档位，见下文 |
| `monitor_models` | string | `general,video` | 需监控的模型，逗号分隔；留空=全部 |
| `schedule_platform` | string | `aiocqhttp` | 推送平台适配器（播报与告警共用） |
| `schedule_target_type` | string | `group` | 推送目标类型：`private`(人) / `group`(群) |
| `schedule_target_id` | string | 空 | 推送目标**纯数字号码**（QQ 号或群号） |

### Cron 表达式（5 段：分 时 日 月 周）

| 示例 | 含义 |
|------|------|
| `0 9 * * *` | 每天 9:00 |
| `0 9,21 * * *` | 每天 9:00 和 21:00 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 9 * * 1` | 每周一 9:00 |

字段顺序：分钟(0-59) 小时(0-23) 日(1-31) 月(1-12) 星期(0-6，0=周日)。

### 阶梯告警逻辑（`alert_levels`）

格式为从高到低的百分比，逗号分隔，例如 `20,10,0`：

1. 余量首次 **≤20%** → 推送一次提醒。
2. 余量继续跌到 **≤10%** → 再推送一次（跨过更低档位）。
3. 余量跌到 **≤0%** → 再推送一次。
4. 在同一档位内持续低额度 **不会重复打扰**。
5. 时段重置、余量回升到所有档位之上后，**自动重置告警状态**，下个周期重新计算。

> 这样既能在额度逐步耗尽时分级提醒，又避免了"一直请求 + 反复刷屏"。轮询间隔建议 30 分钟，足够及时且不会过频访问接口。

## 指令

| 指令 | 说明 |
|------|------|
| `/mmquota` | 查询当前各模型时段剩余额度 |

## 说明

- 接口返回的 `usage_count/total_count` 对按时间计量的套餐恒为 0，故播报只展示**剩余百分比 + 重置时间**，更贴合实际计量方式。
- 定时播报与阶梯告警共用同一组推送目标配置（平台 / 类型 / 号码）。
- API Key 仅保存在 WebUI 配置中，不写入代码，也不会随仓库泄露。
